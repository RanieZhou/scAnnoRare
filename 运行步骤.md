# scAnnoRare 运行步骤

单细胞细胞类型注释与稀有细胞识别多方法评估系统 V1.0 —— 本地启动与使用指南。

系统由三个服务组成，需要分别启动：

| 服务 | 目录 | 端口 | 说明 |
|---|---|---|---|
| Web 后端 | `web-backend` | 8000 | FastAPI + MySQL，提供平台 API |
| Local Agent | `local-agent` | 17890 | 本地计算节点，执行 h5ad 解析与评估 |
| 前端 | `web-frontend` | 5173 | Vue3 界面 |

> **两种使用形态**：
> - **开发/源码模式**（本文档）：分别用 Python/npm 启动三个服务，便于开发调试。
> - **桌面应用模式**：Web 前后端部署在服务器，终端用户安装 **scAnnoRare Agent** 桌面应用
>   （Tauri，自带计算引擎、无需 Python）。打包与分发见
>   [desktop-agent/README.md](desktop-agent/README.md)。

## 一、环境准备

### 1. 基础环境

- Python 3.10 / 3.11（建议）
- Node.js 18+ 与 npm
- MySQL 8.0+

### 2. 数据库

系统使用 MySQL 存储元数据。默认连接配置在 [web-backend/app/db.py](web-backend/app/db.py)，
也可用环境变量覆盖：

```
MYSQL_HOST   数据库主机地址
MYSQL_PORT   3306
MYSQL_USER   数据库用户名
MYSQL_PASSWORD  数据库密码
MYSQL_DB     scannorare
```

> 数据库 `scannorare` 需提前创建；后端首次启动会自动建表并创建默认账号 `admin / admin`。
> 建库命令：`CREATE DATABASE scannorare CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`

## 二、安装依赖

### 1. Web 后端

```bash
cd web-backend
pip install -r requirements.txt
```

### 2. Local Agent（含计算引擎）

```bash
cd local-agent
pip install -r requirements.txt
```

> 首次安装 scanpy / anndata 等生信包耗时较长，请耐心等待。
> GPU 检测为可选项，如需启用，额外安装 `torch` 与 `pynvml`。

### 3. 前端

```bash
cd web-frontend
npm install
```

## 三、启动服务

> 推荐顺序：MySQL → Web 后端 → Local Agent → 前端。
> 三个服务各占一个终端窗口，保持运行。

### 终端 1 —— Web 后端（端口 8000）

```bash
cd web-backend
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

验证：浏览器访问 http://127.0.0.1:8000/docs 出现 API 文档即正常。

### 终端 2 —— Local Agent（端口 17890）

```bash
cd local-agent
python -m uvicorn main:app --host 127.0.0.1 --port 17890
```

验证：访问 http://127.0.0.1:17890/api/v1/local/health 返回 `"status": "healthy"` 即正常。

### 终端 3 —— 前端（端口 5173）

```bash
cd web-frontend
npm run dev
```

验证：浏览器访问 http://localhost:5173 出现登录页即正常。

### 一键启动（Windows）

Windows 下可直接运行脚本同时拉起三个服务：

```powershell
scripts\start_all.ps1
```

或双击 `scripts\start_all.bat`。

## 四、首次使用流程

1. **登录**：浏览器打开 http://localhost:5173，用默认账号 `admin / admin` 登录
   （登录页底部可一键填入）。

2. **配对计算节点**：进入「计算节点配对」页 → 点击「一键在本地生成配对码并授权」
   （或手动输入配对码）→ 右侧出现本地算力与依赖环境即配对成功。

3. **注册数据集**：进入「数据集注册」页 → 填入本地 `.h5ad` 绝对路径 → 「检测 H5AD 数据结构」
   → 选择标签列、设置稀有阈值 → 「分析标签分布并注册数据集」。
   - 测试数据：`data/pancreas_baron.h5ad`（标签列为 `cell_type`）
   - 或用预置 tiny 数据：页面「快速测试指引」一键填入。

4. **配置实验**：进入「实验配置管理」页 → 选择数据集 → 填实验名 → 选评估类型
   （注释评估 / 稀有识别评估）→ 「发布实验基准配置」。

5. **运行评估**：进入「评估与对比」页 → 顶部选择实验 → 填方法名与预测结果 CSV 路径
   → 「提交并在本地节点运行评估」→ 任务完成后自动出现在对比排行榜。
   - 测试 CSV：`local-agent/workspace/predictions/imported_predictions/tiny_predictions.csv`

6. **查看报告**：进入「报告中心」页 → 「在线查看报告」或「下载 HTML」。

## 五、常见问题

**Q：前端能打开但操作报错？**
确认 Web 后端（8000）已启动，且数据库连接正常。

**Q：配对页提示「未检测到本地 Agent」？**
确认 Local Agent（17890）已启动；访问健康检查接口确认。

**Q：配对成功后右侧环境信息不显示？**
多为会话令牌失效（Agent 曾重启）。系统会自动回到等待配对状态，重新配对即可。

**Q：评估任务 failed？**
点「查看运行日志」看 stderr。常见原因：预测 CSV 缺 `cell_id`/`pred_label` 字段、
标签列名不对、cell_id 与数据集完全不匹配。

**Q：后端启动报数据库错误？**
检查 MySQL 是否运行、连接参数是否正确、`scannorare` 库是否已创建。

**Q：端口被占用？**
结束占用进程后重启，或修改启动命令中的 `--port`。
macOS/Linux 查端口：`lsof -ti:8000 | xargs kill -9`
